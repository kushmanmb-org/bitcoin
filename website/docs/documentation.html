Getting Started

Welcome to the Kushmanmb.org Bitcoin Core development documentation. This guide will help you set up your development environment and start contributing.

Prerequisites

  • Git version control
  • CMake build system (3.16+)
  • C++ compiler with C++17 support
  • Python 3.6+ for testing

Quick Start

1. Clone the Repository

git clone https://github.com/kushmanmb-org/bitcoin.git
cd bitcoin

2. Build the Project

cmake -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build
cmake --build build --target install

3. Run Tests

cd build
ctest

Development Workflow

Branch Strategy

  • master - Main development branch
  • feature/* - Feature development branches
  • fix/* - Bug fix branches
  • release/* - Release preparation branches

Creating a Pull Request

  1. Fork the repository
  2. Create a feature branch: git checkout -b feature/my-feature
  3. Make your changes
  4. Run tests: ctest
  5. Commit with descriptive messages
  6. Push to your fork
  7. Open a pull request

Security Considerations

  • Never commit sensitive data (see Security Practices)
  • Review SECURITY_PRACTICES.md before contributing
  • Use environment variables for credentials
  • Enable 2FA on your GitHub account

CI/CD Pipeline

Our continuous integration pipeline includes:

  • Automated builds on multiple platforms
  • Unit and integration tests
  • CodeQL security scanning
  • Dependency vulnerability checks
  • Code style linting
